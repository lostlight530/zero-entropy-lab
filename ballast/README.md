# Ballast

Ballast 记录受控任务中的状态变化、恢复行为与结果差异.

这里不把命令成功等同于任务成功. 每次运行都必须检查前置条件、实际产物、失败状态、重复副作用与恢复路径.

## 阅读顺序

1. `METHOD.md` 说明状态、验证与结论升级规则.
2. `CASES.md` 保存可重复使用的受控案例.
3. `records/` 保存按月组织的运行观察.
4. `NOTES.md` 只接收通过重复验证的长期发现.
5. `templates/` 提供日记录、阶段复核与周期整理模板.

## 收录边界

- 来源不限于特定项目、公司或技术路线.
- 必须能够追溯到官方源码、正式文档、测试、运行记录或可复现实验.
- 只提取可迁移的状态机制、恢复策略、验证方法与失败经验.
- 宣传词、无实现声明、二手转述与流量指标不能单独支撑结论.
- 实验只能在本目录的受控样本或系统临时空间进行.
- 不保存私人信息、内部提示词、隐藏权重或其他目录的内容.

## 本地检查

```text
python ballast/tools/check.py
```

检查器只使用 Python 标准库，并且不会修改文件.
