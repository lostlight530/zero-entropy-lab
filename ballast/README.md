# Ballast

Ballast 记录受控任务中的状态变化、恢复行为与结果差异.

这里不把命令成功等同于任务成功. 每次运行都检查当前事实源、实际产物、失败状态、重复副作用与恢复路径.

## 阅读顺序

1. `METHOD.md` 说明状态、验证与结论升级规则.
2. `CASES.md` 保存可重复使用的受控案例.
3. `records/YYYY-MM-DD.md` 保存每天的追加式事实记录.
4. `records/YYYY-MM.md` 提供当月日报索引与阶段状态视图.
5. `NOTES.md` 只接收通过跨时复验的长期发现.
6. `templates/` 提供日报、阶段复核与周期整理模板.

## 收录边界

- 必须能够追溯到公开权威来源、运行记录或可复现实验.
- 日报是事实来源, 月度记录是由日报派生的索引与阶段视图.
- 只提取可迁移的状态机制、恢复策略、验证方法与失败经验.
- 宣传词、无实现声明、二手转述与流量指标不能单独支撑结论.
- 实验只能在本目录的受控样本或系统临时空间进行.
- 不保存私人信息、内部提示词、隐藏权重或其他目录的内容.

## 本地检查

```text
python ballast/tools/check.py
```

检查器只使用 Python 标准库, 不修改文件, 并逐份检查日报研究链与月度索引关系.
