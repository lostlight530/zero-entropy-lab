### 研究问题

固定前置检查能否在修改发生前区分环境失败与任务失败.

- 使用全新工作副本.

- 网络工具可能采用不同代理路径.
- 临时目录写入能力尚未验证.

### 假设

如果先分别验证身份、远端读取、传输路径和临时写入，环境限制将被明确隔离，不会被误报为任务逻辑缺陷.

### 前置检查

1. GitHub API 身份与权限.
2. 远端默认分支读取.
3. Git 传输路径.
4. Python 临时目录写入.

2. 使用 Git 默认网络路径读取远端.
3. 在默认路径失败后，动态读取当前系统代理并仅注入当前 Git 命令.
4. 使用两个 Python 运行时执行仓库规定的测试入口.
5. 将失败按环境、任务和验证阶段分类.

### 原始观测

- 两个 Python 运行时都能编译目标文件.
- Python 子进程无法在受控工作区中新建测试子目录，相关测试在准备样本阶段停止.
- 云端对应工作流的最新运行结果为成功.

### 后置验证

- 没有把本地权限错误报告为仓库代码失败.
- 没有因环境阻塞修改原仓代码.
- 没有绕过失败测试并宣称全部测试通过.
- 失败目录被识别为本次运行产生的临时状态.

### 反例

- 前置检查过严可能阻止本来不需要临时写入的只读任务.
- 云端成功不能证明所有本地平台都兼容.
- 代理路径成功一次不能证明未来会持续可用.

### 暂时结论

状态为观察. 分阶段前置检查在本次运行中正确隔离了网络路径和临时写入限制，并避免了一次假缺陷判断.

## 2026-07-21

### 研究问题

独立内容校验能否阻止退出码为 0 但产物不完整的受控任务被记为成功.

### 初始状态

- 工作分支基于最新远端 `main`.
- 受控产物路径在实验前不存在.
- 系统临时根不可写, 因此实验限定在 `ballast` 自己的受控样本中.
- 实验只使用 PowerShell 与 .NET 内置能力.

### 可证伪假设

如果退出码和文件存在性不足以证明完成, 那么截断产物会通过浅层检查, 但会被独立长度与摘要校验拒绝. 完整产物应在重复写入后保持等价.

### 控制条件

- 完整与截断路径使用同一目标文件和同一预期字节序列.
- 两条路径都返回退出码 0.
- 独立校验只读取最终产物, 不读取生产步骤的成功信息.
- 每次完整写入使用覆盖语义, 不追加文件或记录.

### 实验

1. 写入预期字节序列的前 12 字节并返回 0.
2. 分别运行文件存在性检查与长度加 SHA-256 摘要检查.
3. 覆盖写入完整产物, 独立校验后保存摘要.
4. 再次覆盖写入完整产物, 比较独立校验和摘要.
5. 删除受控产物并确认路径不存在.

### 原始观测

- 截断写入返回 0.
- 文件存在性检查返回真.
- 独立内容校验返回假.
- 两次完整写入都返回 0, 两次独立内容校验都返回真.
- 两次完整写入的摘要一致, 最终副作用文件数为 1.
- 实验路径完成清理, 验证后有效耗时为 94.561 毫秒.

### 后置验证

- 实验工具对每个预期状态执行断言, 任一差异都会返回失败.
- 受控临时文件在工具退出后不存在.
- 实验工具只在 `ballast` 内创建一个受控样本文件.

### 反例

- 如果只检查文件存在, 截断产物仍会被错误接受.
- 长度与摘要只能证明产物等于已知字节序列, 不能证明未知任务的语义质量.
- 单文件覆盖不代表多文件任务、远端副作用或并发写入也具有相同结果.

### 暂时结论

状态为观察. 在本次受控 B-04 中, 退出码加文件存在性产生了一次假成功判定, 独立内容校验阻止了该判定. 证据不足以升级为候选或长期结论.

### 复验条件

## 2026-07-22

### 研究问题

结构化产物在写入中断但命令返回 0 时, 候选校验与替换失败后的安全停止能否共同阻止假成功和状态污染.

### 初始状态

- 工作分支基于最新远端 `main`.
- 昨日观察使用固定字节序列, 今日改用带状态、项目计数和载荷摘要的清单.
- 目标、候选和备份路径在实验前都不存在.
- 三个受控路径位于 `ballast` 同一目录, 不修改权限或忽略替换错误.

### 公开依据

- [RFC 9110 第 9.2.2 节](https://www.rfc-editor.org/rfc/rfc9110.html#section-9.2.2) 将幂等定义为多次相同请求与单次请求具有相同预期效果, 并要求不能确认语义幂等时避免自动重试.
- [Microsoft ReplaceFileW 文档](https://learn.microsoft.com/en-us/windows/win32/api/winbase/nf-winbase-replacefilew) 说明候选文件替换、同卷限制、删除与 ACL 权限要求以及失败后的多种文件状态.
- 查询日期为 2026-07-22, 只使用标准与官方 API 文档约束实验边界.

### 可证伪假设

如果存在性和结构完整都不足以证明任务完成, 那么中断产物会通过存在性检查但未通过独立校验, 摘要错误的完整候选会通过结构校验但未通过语义校验. 如果验证通过后的替换因权限失败, 任务应保持可判定状态、报告未完成并清理受控路径, 而不是自动重试或宣称成功.

### 控制条件

- 中断、候选和目标使用同一清单格式.
- 中断步骤写入状态、项目计数和一个项目后返回 0.
- 结构校验要求五个唯一字段齐全.
- 语义校验要求完成状态、项目数量和载荷摘要一致.
- 候选、目标和备份位于同一目录, 满足同卷控制条件.
- 替换只尝试一次, 不修改 ACL, 不启用忽略错误选项.

### 实验

1. 写入缺少第二个项目和载荷摘要的中断清单并返回 0.
2. 分别运行文件存在性、结构和语义校验.
3. 写入字段齐全但摘要错误的候选, 验证结构与语义, 未通过时删除候选并检查目标未变.
4. 写入验证通过的候选, 调用 `File.Replace` 并捕获替换结果.
5. 替换失败时检查目标、候选和备份状态, 禁止自动重试并标记未完成.
6. 清理目标、候选和备份, 确认三个路径都不存在.

### 原始观测

- 中断写入返回 0, 文件存在性检查返回真.
- 中断产物的结构校验和语义校验都返回假.
- 摘要错误的候选通过结构校验但未通过语义校验.
- 无效候选被阻止后, 中断目标的文件摘要保持不变.
- 有效候选同时通过结构和语义校验.
- `File.Replace` 未成功, 错误类型为 `UnauthorizedAccessException`.
- 替换失败后, 中断目标和有效候选都保持可验证状态, 未创建备份.
- 任务标记为安全停止, 有效完成返回假.
- 三个受控路径完成清理, 验证后有效耗时为 204.078 毫秒.

### 实验准备失败

- 空备份路径在当前运行时触发非法路径错误 1 次.
- 一处 PowerShell 布尔表达式调用歧义触发参数重复错误 1 次.
- 替换权限错误首次出现时未被工具捕获 1 次, 随后被纳入正式安全停止路径.
- 三次准备失败都在正式实验前停止并完成受控路径清理, 不纳入机制证据.

### 后置验证

- 实验工具分别断言中断拒绝、无效候选拒绝、替换结果、安全停止和清理状态.
- PowerShell 语法解析错误为 0.

### 反例

- 字段齐全且摘要自洽的错误业务值仍可能通过当前语义校验.
- 当前权限错误不能证明其他 Windows 环境中的 `File.Replace` 也会失败.
- Microsoft 文档说明写穿标志不受支持, 本实验不能证明断电后的持久性.
- 替换 API 存在多种失败后文件状态, 当前观察只覆盖权限拒绝这一种.
- 单文件安全停止不能推广到多文件、远端副作用或并发写入.
