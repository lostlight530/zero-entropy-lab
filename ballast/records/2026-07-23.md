# Ballast 日记录 2026-07-23

## 研究问题

Agent 从中断点恢复时, 内部摘要自洽能否证明多文件结果仍对应当前事实源, 当前代际验证能否拒绝旧结果并保持重放幂等.

## 来源依据

查询日期为 2026-07-23.

- [Kubernetes 控制器过期缓解](https://kubernetes.io/blog/2026/04/28/kubernetes-v1-36-staleness-mitigation-for-controllers/) 说明过期缓存会导致错误动作, 协调前应比较资源版本.
- [RFC 9110 第 9.2.2 节](https://www.rfc-editor.org/rfc/rfc9110.html#section-9.2.2) 将幂等限定为重复请求与单次请求具有相同预期效果.
- [W3C PROV-DM](https://www.w3.org/TR/prov-dm/) 区分活动使用与生成的实体, 明确派生路径有助于复现.
- [Microsoft Event Sourcing pattern](https://learn.microsoft.com/en-us/azure/architecture/patterns/event-sourcing) 将追加事件视为权威事实源, 派生视图可重建, 重复交付处理器应幂等.

这些来源支持代际绑定、来源追踪与重放设计, 不直接证明分布式适用性.

## 可证伪假设

路径 A 认为状态完整、文件集合与内部摘要一致即可发布. 路径 B 要求恢复后回读当前事实源, 旧结果即使自洽也应拒绝, 重新同步后的相同输入重放应零写入.

旧结果若未通过内部校验, 就不能攻击路径 A. 当前验证若接受旧结果、拒绝新结果或重放产生写入, 路径 B 被推翻.

## 控制条件

- 任务把每个输入值乘以 2 并生成多文件结果与清单.
- 第一代含 `alpha=2` 与 `beta=3`.
- 第二代保留 `alpha=2`, 把 `beta` 改为 5, 新增 `gamma=7`.
- 中断发生在第一代 `alpha` 输出后.
- 旧路径沿用第一代快照恢复.
- 内部校验只读结果, 当前验证由独立 PowerShell 进程回读事实源.
- 受控文件使用固定前缀, 开始前不存在, 结束后逐文件删除.

## 实验设计

读取第一代并在首个输出后中断, 推进事实源后沿旧快照恢复并生成自洽清单. 比较内部与当前验证, 再清除旧派生物、回读第二代、重建并独立验证. 最后重放当前输入, 比较写入数与包摘要并清理.

## 原始观测

轨迹为 `read_generation_1 → interrupted_after_alpha → source_advanced_to_generation_2 → resumed_from_stale_snapshot → stale_package_rejected_by_current_contract → rebuilt_from_generation_2 → published_generation_2 → replay_noop_after_current_state_check`.

- 旧路径写入 3 次并通过内部校验 `internal_consistency`.
- 独立进程以 `source_generation_mismatch` 拒绝旧结果, 退出状态 1.
- 重新同步写入 4 次, 生成 3 个结果与 1 份清单.
- 独立进程以 `current_contract_satisfied` 接受新结果, 退出状态 0.
- 重放写入 0 次, 再验证通过, 包摘要不变.
- 有效完成、假成功识别与重新同步后恢复均为真.
- 正式重试 1 次, 人工介入 0.
- 清理通过, 验证后有效耗时 5493.638 毫秒.
- 提交前复验得到相同状态、写入数与清理结果, 验证后有效耗时 5039.802 毫秒.

Python 准备原型因当前沙盒新建临时子目录权限失败而未进入实验逻辑, 不纳入机制证据. 正式实验改用 Ballast 固定前缀受控样本并完成清理.

## 独立验证

验证在独立 PowerShell 进程运行, 回读当前事实源并重新计算代际、来源摘要、输出集合、业务值和文件摘要. 限制是生产器与验证器共享脚本与模式, 仍可能共享解析或规则错误.

## 强反例

旧结果具有完整状态、匹配旧清单的文件集合与正确内部摘要, 因此内部校验通过. 相对第二代事实源, 它遗漏 `gamma`、使用旧 `beta` 并声明旧代际, 推翻内部自洽足以证明完成的解释.

## 路径比较

路径 A 用 3 次写入得到错误结果并产生假成功. 路径 B 增加当前回读与独立验证, 拒绝旧结果后用 4 次写入得到正确结果. 当前状态重放先验证后零写入. 较少写入的错误路径不构成速度改进.

## 暂时结论

假成功候选在受控本地文件产物范围内升级为发现. 三个时间窗口分别覆盖截断字节、结构化清单和自洽但过期的多文件结果, 都显示生产路径成功证据不足以证明有效完成.

重新同步后的中断恢复只观察 1 次, 不单独升级长期结论.

## 复验条件

迁移到远端副作用或并发写入, 检查读取与提交间再次变化的竞态. 使用不同实现验证器攻击共享错误. 对非幂等外部操作验证去重与安全停止.

## 事实分层

### 已验证事实

旧结果通过内部摘要但被当前代际拒绝. 新结果通过独立进程检查, 重放零写入且摘要不变, 受控文件完成清理.

### 基于证据的推断

绑定当前事实源和任务后置条件比只验证内部自洽更能阻止过期结果被报告为成功.

### 未验证事项

远端副作用、并发、不可逆操作、开放式语义质量和独立实现验证器未覆盖. Python 准备失败的沙盒残留未作为实验结果验证.

## 指标

- 有效完成、重放一致、幂等通过、中断恢复: 是.
- 假成功: 内部路径 1 次, 当前验证拦截 1 次.
- 正式无效操作 0, 重试 1, 人工介入 0.
- 结果质量: 旧路径错误, 当前绑定路径正确.
- 写入: 旧路径 3, 重新同步 4, 重放 0.
- 验证后的有效耗时: 首次 5493.638 毫秒, 提交前复验 5039.802 毫秒.
