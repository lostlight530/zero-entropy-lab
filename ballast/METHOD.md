# 方法

## 来源闸门

候选材料进入研究前必须通过以下检查.

1. 来源能够公开访问并准确定位.
2. 至少具有官方源码、正式文档、测试、追踪记录或可复现实验之一.
3. 能够提取具体状态转换、验证机制或失败处理，而不是只有愿景.
4. 能够说明机制的代价、失败条件或适用边界.
5. 不依赖热度、厂商规模和二手评价决定价值.

来源集合保持开放. 任何通过闸门的真实实现都可以进入研究，不设置厂商、框架或项目白名单.

## 成功定义

一次任务只有同时满足以下条件才能记为有效完成.

- 前置条件已验证.
- 状态变化在允许范围内.
- 目标产物真实存在且内容有效.
- 后置条件独立通过.
- 重复执行不会产生额外副作用.
- 失败时没有把部分结果报告为完整成功.

退出码、日志中的成功文字和单个文件存在都不能独立证明任务完成.

## 固定研究链

每项研究必须完整记录.

`研究问题 → 可证伪假设 → 控制条件 → 实验 → 原始观测 → 反例 → 暂时结论 → 复验条件`

环境失败、任务失败、验证失败和证据不足必须分别标记.

## 结论状态

- 观察: 只出现一次.
- 候选: 在独立条件下重复出现，但尚未完成跨时复验.
- 发现: 至少经过三个独立实验、跨两个时间窗口并完成反例检查.
- 失效: 新证据推翻原结论，保留原记录并写明原因与日期.

只有发现可以进入 `NOTES.md`.

## 比较维度

- 有效完成.
- 重放一致.
- 幂等通过.
- 中断恢复.
- 假成功.
- 无效操作.
- 重试次数.
- 人工介入.
- 验证后的有效耗时.

这些维度分别记录，不压缩成一个综合分数.

## 记录节奏

- 日记录使用 `templates/daily.md`.
- 阶段复核使用 `templates/weekly.md`.
- 周期整理使用 `templates/monthly.md`.

模板规定内容结构，不规定外部调度方式.
